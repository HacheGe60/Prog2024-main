Todos estos son ejemplos de números en JavaScript. No importa si es negativo o positivo, o si tiene decimales (valores después del punto .). Los llamamos números.

Separador numérico

Puedes representar números más grandes de manera más clara, si lo deseas, con el separador numérico (_). Por ejemplo:

let nb = 1_000 // equivalente a 1000

Esto hace que los números más grandes sean más fáciles de leer. Por ejemplo:

let nb = 1_000_000 // equivalente a 1000000 (1 millón)

El guion bajo _ es eliminado por JavaScript del número. Por lo tanto, 10_00 es equivalente a 1000.

Convertir de número a cadena

Aunque rara vez se usa, puedes convertir un número a una cadena llamando al método .toString().

Tomemos un ejemplo donde tenemos una variable llamada answer con un valor 42;

let answer = 42;

let answerString = answer.toString();

console.log(answerString); // "42"

Prueba

Number.prototype.toString();

Resumen

Devuelve una cadena que representa al objeto Number especificado

Sintaxis

number .toString( [base] )

MDN Parámetro

base: Un entero entre 2 y 36 especificando la base a usar para representar los valores numéricos.

MDN Devuelve

Una cadena que representa al objeto.

MDN Descripción

El objeto Number sobreescribe el método toString y no lo hereda del objeto Object; por tanto no hereda Object.prototype.toString(). Para objetos Number, el método toString devuelve una cadena que representa al objeto en la base especificada.

El método toString comprueba su primer argumento, e intenta devolver una cadena de representación en la base especificada. Para bases superiores a 10, las letras del alfabeto indican numerales mayores de 9. Por ejemplo, para números hexadecimales (base 16), se utiliza de A hasta F.

Si no se da a toString una base entre 2 y 36, se lanza una excepción.

Si no se especifica la base, JavaScript asume la base predefinida, que es 10.

MDN Ejemplos

let howMany = 10;

 

console.log("howMany.toString() is " + howMany.toString()); // muestra "10"

console.log("45 .toString() is " + (45).toString()); // muestra "45"

let x = 7;

console.log(x.toString(2)); // muestra "111"

NaN

A veces puedes encontrarte con NaN, que significa "No es un número". Por ejemplo, si intentas multiplicar un número por una cadena (lo que no debes hacer):

//

"abc" * 4 // NaN

NaN suele ser una señal de que algo está mal en tu código, a menudo se olvidó convertir una cadena a un número. Uno de los casos más comunes es cuando una propiedad de un objeto se evalúa como undefined debido a un error tipográfico y luego se usa como si fuera un número válido (más sobre esto en capítulos posteriores a medida que aprendemos sobre objetos y propiedades de objetos).

Resumen

  • Convertir de un número a una cadena: value.toString()
  • NaN significa "No es un Número" (Not a Number).
  • NaN suele ser un indicio de un error.

Prácticas

Conversión de número a cadena

Completar la función convertNumberToString para que convierta el número que recibe en una cadena.

function convertNumberToString(number) {

   return number.toString();

}

 

// Ejemplos

console.log(convertNumberToString(42)); // "42"

console.log(convertNumberToString(97)); // "97"

console.log(convertNumberToString(11)); // "11"
