Rasumen del capítulo

Una función de devolución es como dejar un número de teléfono para que alguien te llame cuando esté listo. En JavaScript, proporcionas una función de devolución como argumento, que se llama cuando la función principal está lista para enviar el resultado.

En el próximo capítulo, aprenderemos más sobre las funciones de devolución, así como sobre métodos adicionales de arrays que usaremos a lo largo de todo el curso. Estos métodos resultarán ser útiles especialmente una vez que trabajemos con arrays de objetos, que son la estructura de datos más común que se obtiene de las API.

Resumen del capítulo

  • const data = [1, 2, 3] es un array que contiene 3 números.
  • array.length devuelve el número de elementos dentro del array.
  • array .push() te permite agregar la variable x al final del array. También devuelve la nueva longitud del array (después de que se haya realizado el push).
  • Los arrays definidos con const no son constantes porque puedes cambiar los elementos dentro de ellos. Sin embargo, no puedes reasignarlos a otro valor, por lo que siempre serán un array.
  • .forEach(callback) itera sobre cada elemento en un array.
  • Una función de devolución es una definición de función pasada como parámetro a otra función.
  • Siempre comienza con un console.log() dentro de .forEach() para visualizar el cambio de array a elemento de array. (Puedes omitir esto cuando te acostumbres).
  • El método .forEach() tomará tu definición de función y la llamará para cada elemento del array. Cada vez que se llame a la función de devolución, el primer parámetro representará el elemento correspondiente del array.
  • Nombra tus arrays en plural y el elemento del array (dentro de .forEach()) en singular.
  • Asegúrate de colocar correctamente el return dentro de una función que contenga .forEach()
