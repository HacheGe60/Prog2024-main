Array find

En la lección anterior, aprendimos sobre el método .filter() de los arrays. En esta lección, exploraremos el método .find(), que es un poco similar en su funcionamiento.

Comencemos con un ejemplo, esta vez con un array de cadenas:

const names = ["Javier", "Linda", "Ana", "Liz"];

const result = names.find( function(name) {

  return name === 'Ana';

});

console.log(result); // 'Ana'

Prueba

Cuando llamas al método .find(callback) en un array, obtendrás el primer elemento que coincida con la condición que especifiques. Si no se encuentran elementos, obtendrás undefined.

La condición que especificamos aquí es que el name debe ser igual a "Ana".

Entonces, el método .find(callback) llamará a la función de devolución que proporcionaste para cada elemento del array hasta que una de las funciones de devolución devuelva true. Cuando esto suceda, dejará de llamar a las funciones de devolución restantes y te devolverá el elemento para el cual la función de devolución devolvió true.

En nuestro ejemplo anterior, aquí está la función de devolución:

function(name) {

  return name === 'Ana';

}

que se llama para name = "Javier" (primer elemento del array). Sin embargo, la función de devolución devolverá false porque name === "Ana" devuelve false. Por lo tanto, la función de devolución se llamará nuevamente con el siguiente valor de name. Nuevamente devolverá false porque name = "Linda" es distinto a Ana. Esta vez, name = "Alex". La función de devolución devolverá true porque name === "Ana" (la condición dentro de la función de devolución) devuelve true. Por lo tanto, el método .find() se detiene y te devuelve ese elemento, que es "Ana".

Veamos otro ejemplo, esta vez con un array de números:

const numbers = [9, 5, 14, 3, 11];

const firstNumberAboveTen = numbers.find( function(number) {

  return number > 10;

});

console.log(firstNumberAboveTen); // 14

Prueba

Observa cómo, incluso si hay 2 números que satisfacen la condición, el método .find() se detiene en el primero que satisface la condición.

Esto nos llevará a la siguiente sección, que es .filter() vs .find(). ¿Cuáles son las diferencias?

.filter() vs .find()

¿Cuál es la diferencia entre .filter() y .find()?

La diferencia tiene que ver con el tipo de retorno de estos 2 métodos:

  1. El método .filter() siempre devuelve un array.
  2. El método .find() devuelve el primer elemento del array que coincide con la condición de la función de devolución o undefined.

Veamos algunos ejemplos:

const numbers = [9, 5, 14, 3, 11];

 

// .filter() SIEMPRE devuelve un array

numbers.filter(function(number) {

  return number >= 12;

}); // 14

 

// .find() devuelve el primer elemento que coincide con la condición o undefined

numbers.find(function(number) {

  return number >= 12;

}); // 14

Observa cómo .filter() devuelve un array, incluso si solo hay 1 elemento que coincida con tu condición. Por el contrario, el método .find() devolverá el primer elemento que coincida con la condición.

.filter() siempre devuelve un array. Incluso si no coincide ningún elemento.

Ahora veamos un ejemplo donde ningún elemento satisface la condición:

const numbers = [9, 5, 14, 3, 11];

 

// .filter() SIEMPRE devuelve un array

numbers.filter(function(number) {

  return number >= 15;

}); // []

 

// .find() devuelve el primer elemento que coincide con la condición o undefined

numbers.find(function(number) {

  return number >= 15;

}); // undefined

Observa cómo .filter() devuelve un array vacío y .find() devuelve undefined.

.find(callback) puede devolver undefined. Es posible que debas envolver su resultado en una condición if para evitar errores inesperados si terminas llamando a un método en su resultado.

Resumen

  • El método .find() devuelve el primer elemento que coincida con la condición impuesta. Si no se encuentran elementos, devuelve undefined.
  • El método .filter() siempre devuelve un array. Incluso si está vacío.

Prácticas

Encontrar un determinado año

Completa la función getYear para que devuelva el searchYear (pasado como segundo parámetro) cuando se encuentre en el array. De lo contrario, debería devolver undefined.

Echa un vistazo al uso de ejemplo y la salida esperada para comprender mejor cómo debería funcionar la función.

function getYear(years, searchYear) {

  return years.find(function (year) {

    return year=== searchYear

  });

}

 

// Ejemplo

console.log(getYear([1991, 1992, 1993, 1994, 1995], 1995)); // 1995

console.log(getYear([1991, 1992, 1993, 1994, 1995], 1990)); // undefined

Prueba

Años impares

Completa la función getOddYears para que devuelva todos los años que son impares del parámetro years que recibe.

function getOddYears(years) {

  return years.filter(function (year) {

    return year % 2 !== 0;

  });

}

 

// Ejemplo

console.log(getOddYears([2019, 2020, 2021])); // [2019, 2021]

console.log(getOddYears([2000, 2015, 2018, 2020])); // [2015]
