Objetos

Un objeto es un tipo de dato que te permite agrupar varias variables en una sola variable que contiene claves y valores. Esto se utiliza a menudo para representar o describir una entidad. Por ejemplo, una persona, un usuario, un producto, etc.

Así es como puedes crear un objeto:

const user = {

  id: 1,

  firstName: 'Sam',

  lastName: 'Doe',

  age: 20,

};

En JavaScript, la convención recomendada es usar camelCase para los nombres de propiedades (por ejemplo, firstName en lugar de first_name)./p>

Leer el valor de una propiedad

Para leer el valor de una propiedad en un objeto, puedes usar la notación de punto de la siguiente manera:

const user = {

  id: 1,

  firstName: 'Sam',

  lastName: 'Doe',

  age: 20,

};

  

user.id;// 1

user.firstName;// 'Sam'

user.isAdmin; // undefined (no existe la propiedad 'isAdmin')

Es una buena idea visualizar los objetos con los que estás trabajando registrándolos en la consola. Sin embargo, los objetos anidados no se expanden en la consola de este curso. Puedes usar la consola de tu navegador o, preferiblemente, hacer un console.log directo del objeto anidado.

Más adelante en este curso, veremos cómo leer el valor de una propiedad dinámica.

Actualizando el valor de una propiedad

También puedes actualizar el valor de una propiedad usando la misma notación de punto seguida de un signo igual:

const user = {

  id: 1,

  firstName: 'Sam',

  lastName: 'Doe',

  age: 20,

};

  

user.lastName = 'Blue';

user.age = user.age + 1; // 21

console.log(user); // { id: 1, firstName: 'Sam', lastName: 'Blue', age: 21 }

Puedes actualizar el valor de una propiedad de un objeto definido con const porque const no significa que la variable sea constante, solo significa que no puedes reasignarla. Por lo tanto, la variable siempre es un objeto, pero su contenido (propiedades) puede cambiar.

Resumen

  • Un objeto es un tipo de dato que te permite agrupar varias variables en una sola variable que contiene claves y valores.
  • En JavaScript, la convención recomendada es usar camelCase para los nombres de propiedades (por ejemplo, firstName en lugar de first_name).
  • Para leer o actualizar el valor de una propiedad, puedes usar la notación de punto.

Prácticas

Detalle de productos

Completa la función getProductDetails de manera que devuelva un objeto con las siguientes propiedades:

  • id: Un número que representa el ID del producto.
  • title: Una cadena de texto que representa el título del producto.
  • inStock: Un booleano que representa si el producto está en stock o no.

Asigna a estas propiedades cualquier valor que desees, siempre que coincida con el tipo de retorno esperado.

function getProductDetails() {

  return {

    id: 1,

    title: 'Product 1',

    inStock: true,

  };

};

  

console.log(getProductDetails()); // { id: 1, title: 'Product 1', inStock: true }

Completa la función getWeather de manera que devuelva la cadena de texto:

Actualmente hacen X grados en Y

Donde X es reemplazado por el valor y Y es reemplazado por el nombre de la ciudad. Estos valores se proporcionan como un parámetro city que es un objeto.

Aumentar la edad

Completa la función incrementAge de manera que devuelva el objeto person con la edad incrementada (suma 1 al valor existente).

Ten en cuenta que solo hemos proporcionado un ejemplo de uso en este desafío, pero tu código debería funcionar para cualquier objeto similar con el que lo probemos.

function incrementAge(person) {

  person.age++;

  return person;

};

  

const person = {

  firstName: 'Jennifer',

  lastName: 'Doe',

  age: 18,

};

  

console.log(incrementAge(person)); // { firstName: 'Jennifer', lastName: 'Doe', age: 19 }

Clima I

Una vez que aprendamos sobre la API Fetch, ¡podrás escribir la llamada fetch a una API de clima (demo)!

Para este desafío, ya estamos llamando a la API del clima por ti en index.js. Tienes que completar las siguientes funciones en weather.js:

  • getCountry: Devuelve el país para el que se muestra el clima.
  • getIcon: Devuelve el ícono del clima.
  • getTemperature: Devuelve la temperatura.

Ten en cuenta que estas 3 funciones reciben un objeto data que es una instantánea de una respuesta proveniente de una API real (weatherstack.com).

No puedes resolver este desafío sin comenzar con un console.log(data) en cada función. Tienes que explorar el objeto y ver dónde puedes obtener la propiedad que estás buscando.

Ah, y hemos deshabilitado el autocompletado para que puedas resolverlo tú mismo ;) ¡Buena suerte!

export function getCountry(data) {

  console.log(data.location.country);

  return data.location.country;

};

export function getIcon(data) {

  console.log(data.current.weather_icons[0]);

  return data.current.weather_icons[0];

};

export function getTemperature(data) {

  console.log(data.current.temperature);

  return data.current.temperature;

};
