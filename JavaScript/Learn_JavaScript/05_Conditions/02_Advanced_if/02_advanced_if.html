If avanzado

A veces es posible omitir el else. Veamos un ejemplo:

function canVote(age) {

  if (age >= 18) {

    return true;

  } else {

    return false;

  }

}

Dado que esta función está realizando dos acciones diferentes basadas en el resultado de la condición if y su opuesto (else), podemos reescribirla omitiendo la palabra clave else.

function canVote(age) {

  if (age >= 18) {

    return true;

  }

  return false;

}

Estas dos funciones tendrán el mismo resultado exacto. Eso se debe a que la palabra clave return detendrá la función con el resultado. Por lo tanto, cuando la edad sea mayor o igual a 18, la función devolverá true y el resto del código no se ejecutará.

Sin embargo, cuando la edad es menor que 18, entonces el código dentro de la condición if no se ejecutará. Por lo tanto, la única línea que se ejecutará es la última, que es return false.

Aprovecharemos este consejo más adelante en este curso para aprender sobre un patrón común llamado "early return.

Nota de legado

Una nota rápida sobre el legado. Si encuentras == (igual doble) en JavaScript, intenta reemplazarlo con === (igual triple).

El operador de igualdad doble == realiza algunas conversiones que no esperarías. Siempre usa === en su lugar.

Siempre usa === cuando compares 2 valores en JavaScript.

Si deseas saber más por qué no se recomienda ==, entonces consulta esta comparación:

console.log('2' == 2);

Prueba

¿Esto retornaría verdadero o falso?

Retornaría true porque JavaScript intentará convertir ambos valores al mismo tipo de dato. Por favor, no consideres esto una "característica". En su lugar, debes evitarlo y siempre usar === (igual triple), que solo devuelve true si ambos lados son del mismo tipo de dato y tienen el mismo valor.

Resumen

  • Cuando tienes una condición if/else que devuelve dos resultados diferentes, es posible omitir la palabra clave else.
  • Siempre usa === (igual triple) cuando compares 2 valores en JavaScript.

Prácticas

Obtener la próxima edad II

Completa la función getNextAge para que devuelva la edad del próximo año (sumando 1 a la edad actual).

Ten en cuenta que la edad es proporcionada por el usuario en un cuadro de texto (que puedes probar en la pestaña "NAVEGADOR ").

Sin embargo, cuando el cuadro de texto está vacío, la función devuelve NaN. Necesitamos arreglar eso y hacer que muestre 0 en lugar de NaN.

export function getNextAge(age) {

  if (age === '' ) {

    return 0;

  }

  return age + 1;

}
